+Add-on
Self Check-in Module
Available from Month 6 · Premium add-on · Requires PMS integration

A fully automated self check-in flow for guests — no front desk interaction needed. Built with strict validation logic to protect hotels from early arrivals, name mismatches, unpaid balances, and unready rooms. Staff are always kept in the loop via WhatsApp notification at the point of key handover.

Check-in Validation Flow
1
Name Match Verification
The guest enters their name. The system checks it against the booker name on the PMS reservation. Partial matches (e.g. middle names) are flagged for staff review rather than hard-blocked.
✕ Hard block if names do not match
2
Check-in Time Window
The system checks the hotel's configured check-in time (e.g. 3:00 PM). If the guest attempts to check in before this window, they are placed in a holding state — not rejected. They receive a message with the earliest available time and can re-attempt automatically.
⏳ Held with message if too early — not hard blocked
3
Outstanding Payment Check
If the reservation shows an outstanding balance, the guest is prompted to settle via Stripe payment link before proceeding. Check-in is gated until payment is confirmed in real time.
💳 Gated — payment required before proceeding
4
Guest ID Verification
The guest uploads a photo of their passport or driving licence. Stored securely and flagged to hotel staff for review. Check-in continues in parallel — staff can action any concern post-capture.
📋 ID captured — staff notified to review
5
Room Ready Status
The system checks the housekeeping status of the assigned room via the PMS. If not yet marked clean, the guest receives a friendly message with an estimated ready time and is notified automatically once the room is cleared.
🧹 Held with ETA if room not ready — auto-notified when clear
✓
All Clear — Check-in Complete
All checks passed. Guest receives their room number and a personalised welcome message. Staff receive a WhatsApp notification to prepare and hand over the physical key. Reservation is marked checked-in on the PMS automatically.
🗝 Staff notified via WhatsApp to hand over key

05 — Technology
How it's built
Frontend
React + Tailwind
Fast, mobile-optimised guest app and admin panel. PWA for app-like feel without the App Store.
Backend
Node.js + Supabase
Scalable API, real-time database for live request updates, built-in auth for hotel admin access.
AI Layer
Anthropic Claude API
Powers the AI concierge. Hotel-specific context injected per property. Handles escalation to WhatsApp gracefully.
Messaging
WhatsApp Business API
Routes guest requests to hotel staff. Sends automated welcome messages on check-in. Zero friction for guests.
Payments
Stripe
SaaS subscription billing, invoicing for the mini-PMS, and payment link generation for hotel use.
PMS Layer
Unified Integration Hub
Single internal API that normalises data from Mews, Cloudbeds, Opera et al. Add new PMS without rebuilding.
